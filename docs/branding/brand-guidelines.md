# Brand Guidelines

This document defines the official visual identity guidelines for Steam Media Pack.

All visual materials should follow these guidelines to ensure a consistent and recognizable brand.

---

# Brand Values

Steam Media Pack is built around the following principles:

- Simplicity
- Consistency
- Modern Design
- Accessibility
- Open Source
- Community

---

# Visual Identity

The visual identity should communicate:

- Clean interfaces
- Technical quality
- Reliability
- Organization
- Ease of use

Avoid visual clutter and unnecessary decorative elements.

---

# Logo Usage

Always use the official logo.

Do not:

- stretch the logo;
- rotate the logo;
- change its proportions;
- add effects;
- alter its colors.

Refer to the Logo documentation for approved versions.

---

# Color Palette

Use only the official project colors.

Primary, secondary, and accent colors are defined in:

- Colors

---

# Typography

Use the official project fonts consistently across documentation and promotional materials.

Refer to:

- Typography

---

# Icons

Icons should be simple, consistent, and visually aligned with the overall design.

Refer to:

- Icons

---

# Documentation

All documentation should:

- use a clean layout;
- follow consistent spacing;
- use official colors;
- use official typography;
- include high-quality screenshots.

---

# Screenshots

Screenshots should:

- be captured at native resolution;
- avoid unnecessary desktop elements;
- represent the latest version of the project.

---

# Templates

Official templates should be used whenever possible for:

- GitHub images;
- documentation;
- presentations;
- announcements;
- social media.

---

# Consistency

Visual consistency is more important than creativity.

Every new visual asset should reinforce the project's identity rather than introduce a different style.
