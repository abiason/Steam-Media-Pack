# ADR-0001 — Project Structure

- Status: Accepted
- Date: 2026-07-23

## Context

As the project grew, documentation, assets and source code began to evolve independently.

A clear directory structure was required to improve maintainability and contributor onboarding.

## Decision

The repository is organized into the following top-level areas:

- Source Code
- Documentation
- Assets
- GitHub Configuration

Documentation is divided into:

- User Guide
- Developer Guide
- Reference
- ADR
- Assets

## Consequences

### Positive

- Better organization
- Easier onboarding
- Easier navigation
- Scalable documentation

### Negative

- Initial migration effort
